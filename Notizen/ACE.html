Feedforward Control

  • Is a transfer element (like the plant itself).
  • Generates control input signal \(u(t)\) s.t. \(y(t)\) follows the reference signal \(w(t)\) as closely as possible.
  • The controller acts "blind".
  • Can use information from disturbance to correct the control signal.
  • The primary goal is good tracking behavior, i.e. to adapt \( y \) to changing reference \( w \).
Function Name
\( w(t) \) Reference
\( u(t) \) Input
\( z(t) \) Disturbance
\( y(t) \) Output

Feedforward Controller: The targeted influencing of \( y \) with negligible or measurable disturbance \( z \) is carried out by a feedforward controller.

Feedback Control

(sometimes called closed-loop control or control loop)

  • The output variable is measured, compared and corrected via a controller (in case of deviation).
  • The measuring device/element detects the control output variable \( y \) and generates \( y'(t) \) as equivalent as possible to \( y(t) \).
  • \( y(t) \) may be falsified by measuring noise \( n(t) \).
  • The measuring noise is regarded as a component of the disturbance \( z(t) \) and not considered separately.
  • Occasionally the measuring element is accepted as ideal (e.g. \( y'(t)=y(t) \)), and is left out in the block diagram.
  • The nominal-actual value comparison generates the control error: \[ e(t)=w(t)-y'(t) .\]

The primary objective of feedback control

is to reduce the impact of

  1. of disturbances and
  2. of inaccuracies of the plant model.
  3. Secondary objective: improve the tracking behavior
Reference signal \( w(t) \) Externally specified setpoint signal for the control output variable \( y(t) \).
Control error \( e(t) \) Results from comparing the reference signal with the measured control output signal and should be kept small.
Controller/Feedback controller Transfer element which generates the control input signal \( u \) from the control error \( e \), so that \( y \) follows \( w \) as far as possible.
Measuring Element/Measuring Device Detects the controlled variable \( y \) by means of a sensor and generates a signal \( y'(t) \) as equivalent as possible to \( y(t) \)

Feedforward + Feedback = Two-Degrees-of-Freedom Control

Feedback Control Feedforward Control
Purpose Respond to difference between the actual output and the desired output (the error). Acts based on the known input or reference before the system output reacts.
Advantages Naturally rejects disturbances and compensates for modeling inaccuracies because it continuously corrects deviations. Can shape the system response (tracking behavior) effectively when the system model is accurate. For example, it can anticipate how the output should change and act proactively.
Limitations The systems response is reactive, so it may introduce delays or overshoot. Does not reject disturbances or unmodeled dynamics because it doesn't use feedback from the output.

If no disturbance occurs: \[ \begin{split} z=0 \implies & y=w \\ \implies & e=0 \\ \implies & \text{feedback controller remains inactive.} \end{split} \] If a disturbance occurs while \( w \) is zero/constant over a long period of time, the fault is fought by the feedback controller alone.

Often the trajectory generation is included into the feedforward block and the two degrees of freedom structure.

2. Models

Toricell Law

For a tank the volume outflow \( q_\text{out}\) is related to the fill level \( h \), where \( a \) is the cross section area of the drain pipe and \( g \) is the acceleration due to gravity. \[ q_\text{out}(t) = a \sqrt{2gh(t)} .\]

Proportional Variables

To simplify matter, often a proportionality is assumed. Example: The volume inflow \( q_\text{ in } \) is proportional to the angular position \( \phi \) and proportional to an inflow velocity \( v_\text{ in } \). \[ q_\text{in}(t)= \underbrace{ K_Q }_{ \begin{split}&\text{ Proport. }\\ & \text{ factor } \end{split} } \cdot \phi(t) \cdot v_\text{ in }(t) .\]

State-Space Representation

Many dynamic systems can be represented by equations generally formulated with a model consisting of: \[ \begin{split} & \begin{split} & \text{ state differential } \\ & \text{ equations } \\ \end{split} \begin{cases} \dot x_1 = f_1(x_1 ... x_n, z, u) \\ ~~~~~... \\ \dot x_n = f_n(x_1 ... x_n, z, u) \\ \end{cases} \\ &\\ & \text{ output equation } ~~~~~~y=g(x_1,...,x_n) \\ \end{split} .\] It is called a state-space model/representation, if the time responses \( x_1(t),...,x_n(t) \) are uniquely determined for know input signals \( u(t) \) and \( z(t) \) and given initial values \( x_1(0),...,x_n \). These \( n \) time responses together form the trajectory

State Variables
(usually as a vector)		 \( x_1,...,x_n = x(t) = \begin{bmatrix} x_1(t)\\...\\x_n(t) \end{bmatrix} \)
Input Signals \( u(t), z(t) \)

The state differential equations of the output equation can also be written using vectors \[ \dot {\mathbf{ x }} = \mathbf{ f } ( \mathbf{ x } ,z,u ) ,\] \[ y=g( \mathbf{ x } ) .\] Where the vector \( \mathbf{ f } \) comprises \( f_1,...f_n \). If the right hand side of these equations consist exclusively linear combinations of the variables involved, \[ \begin{split} \dot x_1 &= a_{11}x_1+...+a_{1n}x_n+e_1z+b_1u, \\ & ... \\ \dot x_n &= a_{n1}x_1+...+a_{nn}x_n+e_nz+b_nu, \\ \\ y &= c_1x_1+...+c_n x_n, \end{split} \] then the (constant) coefficients can be summarised in vectors \( \mathbf{ e } , \mathbf{ b } , \mathbf{ c } ^{ T } \) and a matrix \( \mathbf{ A } \). \[ \begin{split} \underbrace{ \begin{bmatrix} \dot x_1\\...\\\dot x_n \end{bmatrix} }_{ \dot{ \mathbf{x}}(t) } = \underbrace{ \begin{bmatrix} a_{11} & ... & a_{1n}\\ ... & & \\ a_{n1} & & a_{nn} \end{bmatrix} }_{ \mathbf{ A } } \underbrace{ \begin{bmatrix} x_1\\...\\x_n \end{bmatrix} }_{ \mathbf{ x } (t) } + \underbrace{ \begin{bmatrix} e_1\\\\e_n \end{bmatrix} }_{ \mathbf{ e } } z(t)+ \underbrace{ \begin{bmatrix} b_1\\\\b_n \end{bmatrix} }_{ b } u(t)~~~~ &~~~~ \text{ short } \dot{ \mathbf{ x } } = \mathbf{ Ax } + \mathbf{ e } z + \mathbf{ b } u\\ y= \underbrace{ \begin{bmatrix} c_1&...&c_n \end{bmatrix} }_{ \mathbf{ c } ^{ T } } \underbrace{ \begin{bmatrix} x_1\\...\\x_n \end{bmatrix} }_{ \mathbf{ x } (t) } ~~~~~~~~&~~~~ \text{ short } y= \mathbf{ c } ^{ T } \mathbf{ x }. \\ \end{split} \] In rare cases, input signals can pass directly through to output \(y\). In this case, \[ y=g(x_1,...,x_n) ~~~~~\text{ becomes } ~~~~~y=g(x_1,...,x_n,z,u)= \mathbf{ c } ^{ T } \mathbf{ s } + d_1z + d_2u .\] If there is no disturbance \(z\) to be taken into account, \(\dot{ \mathbf{ x } }\) is simplified to \[ \dot{ \mathbf{ x } }= \mathbf{ Ax } + \mathbf{ b } u .\]
